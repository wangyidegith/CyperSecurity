静态库在哪？
    一个C程序的目录结构一般是这样的：include lib main.c src
    静态库一般被放在lib下



如何制作一个静态库呢？
    ①得到需要制作静态库的.c文件的.o文件：gcc   -o   *.o   -c   *.c
    ②将.o文件制作为静态库：ar   rcs   lib静态库名.a   *.o



如何使用静态库？
1 把静态库的头文件放到main.c中；
2 
    在编译main.c文件时，以【-L 静态库所在目录 -l 静态库名】（也可以没有空格，写成【-L静态库所在目录 -l静态库名】）的方式将静态库链接至目标文件main中：
	gcc   -o   mian   main.c   -Llib   -l静态库名



用法解析：
1 一般原则：main.c中一般不包括.h这是为了良好的文件关系，.h就是函数原型声明、结构体定义、宏定义，当然也有在.h里放其他.h和.c（较少）的。
2 如果你不想别人看见你得源码就给他一个静态库和一个头文件。
总之，一定要理解：.h和.c之间没有必然的联系，有联系的是.o之间，.h只是声明而已。
