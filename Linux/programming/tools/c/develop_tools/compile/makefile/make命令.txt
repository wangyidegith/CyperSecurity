通过规则.txt我们知道makefile文件中可以有多个基本单元（一个基本单元对应一个目标），那么如何执行这些基本单元呢？make

make程序的基本逻辑：
当我们在makefile所处目录下执行make命令时，默认为执行第一个目标，终极目标的依赖==小目标，当makefile在目录下找不见依赖
时，makefile遵照自上而下的原则，便会往下找。

按照这个基本逻辑：
我们在写makefile的时候必须将终极目标放在最上头，小目标放在下面。

除此以外：
我们还可以写一些伪目标，因为有些命令比如rm这样的命令并不会像gcc、mkdir、touch这些命令一样会生成文件，
那么我们如何执行这些伪目标呢？
make 目标名



实际上我们可以使用【make 目标名】执行任何目标，如果不加目标名只用make的话，那就默认为执行第一个目标了
