i-node：文件在硬盘上的索引
	一个文件在硬盘上 == i-node + block
	i-node记录了文件的各种属性（ll）和block的索引
	block是硬盘上具体存储文件信息的地方



文件名（ln、cp）：文件在操作系统中的索引
Linux是怎样管理硬盘的？——维护一个文件系统比如ext4，硬盘中会存储一个可以在计算机开机后载入内存的目录树结构，该树的节点是一个又一个的目录项，目录项存储着文件名与i-node的映射关系
i-node作为文件在硬盘上的索引有且只有一个，但是文件名也就是硬链接（如何理解硬链接：一级指针）可以有很多个
关于硬链接，主要是理解清楚它是一级指针就行了，这样就理顺了它和软链接、cp的区别，软链接是二级指针，cp是普通的非指针类型对象



文件描述符：文件在进程中的索引
”Linux下一切皆文件“实际上说的就是fd的机制，Linux将套接字等不是真正文件的底层资源抽象为了一个可以用fd标识的接口，
让程序员可以用熟悉的文件API操作这些底层资源，但是这些底层资源并一定是传统的存储在硬盘上的文件，它们可能在硬伤有存储，但不是存真正的信息
